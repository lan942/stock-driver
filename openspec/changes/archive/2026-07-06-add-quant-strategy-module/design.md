## Context

当前系统已有 StockDaily 数据（open/high/low/close/volume/turnover_rate 等）和技术指标计算引擎（indicator_engine 支持 MA/MACD/RSI/KDJ/BOLL/ATR/ADX）。需要在此基础上构建一套完整的量化策略模块，实现每日选股推荐 + 持仓管理 + 资金管理。

约束条件：
- A股 T+1：今日买入的股票最早明日才能卖出
- 数据仅包含日线级别 OHLCV，无分钟/分时数据
- 交易价格为日线级别的开盘价/收盘价（无法盘中实时交易）

## Goals / Non-Goals

**Goals:**
- 构建独立、可插拔的策略引擎，不耦合现有模块
- 基于多因子评分模型，每日对所有股票打分，选出最优 N 只
- 输出明确的买入建议（股票代码、建议买入价、目标卖出价、止损价）
- 管理持仓生命周期：开仓 → 监控 → 平仓
- 支持可配置的参数（初始资金、最大持仓、仓位比例、止盈止损比例）
- 通过 REST API 和前端页面暴露策略结果

**Non-Goals:**
- 不处理集合竞价或盘前交易
- 不做实时盘中的动态调仓
- 不处理分红/配股/除权除息
- 第一版不包含 backtrader 回测集成（后续可扩展）
- 不做行业轮动或板块筛选

## Decisions

### 1. 策略引擎采用多因子评分模型

**方案**：每个股票基于多个因子计算综合评分，取 TOP N。
**替代方案**：规则触发式（如 MACD 金叉买入）。多因子评分更灵活、可调参、易扩展。
**因子组成**：
- 趋势因子（权重 30%）：短期（5日）MA 与长期（20日）MA 的乖离率
- 动量因子（权重 25%）：RSI（14日）标准化到 0~1
- 成交量因子（权重 20%）：当日成交量 / 20日均量
- 反转因子（权重 15%）：近3日涨跌幅（均值回归信号）
- 波动率因子（权重 10%）：ATR 标准化

每个因子输出 0~1 分，加权求和得总分。

### 2. 期望收益驱动的止盈止损设计

**核心问题**：仅靠日线 OHLCV 数据做选股，胜率预期在 40%~50%。如果止盈=止损（对称盈亏比），期望收益为负。必须设计非对称盈亏比。

**方案**：
- 用户设定 `target_annual_return`（如 15%）
- 系统基于预期胜率（~45%）反推所需的盈亏比：
  - 期望每笔收益 E = win_rate × stop_profit_pct - (1-win_rate) × stop_loss_pct
  - 设定 stop_profit_pct / stop_loss_pct ≥ 2:1（如 +6% / -3%），确保 E > 0
- 止盈止损比例可由用户直接配置，也可选择"根据目标收益自动计算"

**默认值**：
- `stop_profit_pct`：0.06（+6%），基于 2:1 盈亏比，45% 胜率下 E≈1% /笔
- `stop_loss_pct`：0.03（-3%）
- 用户可覆盖这些值，系统会计算并展示预期年化收益

### 3. 买卖时机与价格锚定

- **推荐时间**：每日收盘后（T日），基于当日收盘数据生成 T+1 日买入推荐
- **买入价格**：T+1 日开盘价（open），设置限价 = T 日收盘价的 101%（允许 1% 溢价追入）
- **卖出价格**：目标止盈价 = 买入价 × (1 + stop_profit_pct)，止损价 = 买入价 × (1 - stop_loss_pct)
- **实际卖出**：当 T+N 日收盘价 >= 目标价或 <= 止损价时触发平仓
- **持仓超时**：若 `max_hold_days` 个交易日内未触发止盈止损，以当日收盘价强制平仓

### 4. 资金分配：等权重模型

每只股票分配资金 = 总可用资金 × position_ratio
如果可用资金不足以买入 100 股（A股最小交易单位），跳过该推荐。

### 5. 实际收益跟踪与目标对比

系统持续跟踪：
- 已实现收益（已平仓交易的总盈亏）
- 当前持仓浮动盈亏
- 年化收益率 = 累计收益率 / (运行天数 / 365)

前端展示时与实际收益与目标收益（target_annual_return）并排对比，用颜色标记达标/未达标。

### 6. T+1 合规

- 买入日 = T+1，卖出日 >= T+2
- 持仓记录标注 `buy_date`，所有卖出判定必须确保 `sell_date > buy_date`
- 前端展示持仓时标注"T+1 锁定中"状态

### 7. 模块边界

- `strategy_config.py`：配置读写（DB 表）
- `strategy_engine.py`：因子计算 + 评分排序 + 生成推荐
- `position_manager.py`：持仓 CRUD + 卖出判定 + 资金跟踪 + 收益统计
- `strategy.py`（API）：路由注册 + 请求响应
- `strategy.py`（model）：StrategyPosition, StrategyTransaction, StrategyCash, StrategyConfig 四张表

### 8. 无需引入 backtrader

当前 indicator_engine 已封装了 backtrader 指标计算。策略引擎直接调用 indicator_engine 获取指标值，不再额外实例化 Cerebro。后台回测功能后续迭代再加。

## Risks / Trade-offs

- [Risk] 胜率 40%~50% 是经验估计，实际可能更低 → 通过 2:1 以上盈亏比确保正期望；用户可手动调参
- [Risk] 多因子模型的参数是经验值，不一定适配所有市场环境 → 将因子权重放在配置中，允许用户调参
- [Risk] 仅用日线数据可能错过盘中机会 → 日线策略的固有限制，Non-Goal 已明确
- [Risk] A 股涨跌停限制可能导致限价单无法成交 → 策略排除当日涨跌停的股票
- [Risk] 低胜率意味着连续亏损的心理压力 → 前端展示期望收益数学原理，帮助用户理解
- [Risk] 数据量不足时因子计算失真 → 要求股票至少有 60 个交易日数据才纳入评分
